# Week 4 - Java 데이터 타입과 연산자 학습

## 오늘의 학습 내용

### 1. Java 자동 타입 변환 (Type Casting)
- **개념**: 작은 타입에서 큰 타입으로 자동 변환
- **순서**: byte → short → long → float → double
- **예제 파일**: `TypeCastingExample.java`
- **특징**: 명시적 캐스팅 없이 자동으로 변환되며, 데이터 손실 없음

### 2. Java 강제형변환 (Force Casting)
- **개념**: 큰 타입에서 작은 타입으로 강제 변환
- **주의사항**:
  - double → int: 소수점 부분이 버려짐
  - int → byte: 범위 초과 시 오버플로우로 데이터 손실
- **예제 파일**: `ForceCastingExample.java`
- **위험성**: 데이터 손실이 발생할 수 있으므로 주의 필요

### 3. Java 비트 연산 (Bitwise Operations)
- **연산자**:
  - AND (&): 두 비트가 모두 1일 때 1
  - OR (|): 두 비트 중 하나라도 1일 때 1
  - XOR (^): 두 비트가 다를 때 1
  - NOT (~): 비트를 반전 (0→1, 1→0)
- **예제 파일**: `BitwiseOperationsExample.java`
- **활용**: 플래그 처리, 마스크 연산 등에 사용

### 4. Java 시프트 연산 (Shift Operations)
- **연산자**:
  - 왼쪽 시프트 (<<): 비트를 왼쪽으로 이동, 값 × 2^shift
  - 오른쪽 시프트 (>>): 비트를 오른쪽으로 이동, 부호 유지, 값 ÷ 2^shift
  - 부호 없는 오른쪽 시프트 (>>>): 비트를 오른쪽으로 이동, 부호 무시
- **예제 파일**: `ShiftOperationsExample.java`
- **중요 차이**: 음수에서 >>는 부호를 유지하지만 >>>는 0으로 채워 양수로 변환

## 학습 목표
- Java의 기본 데이터 타입 간 변환 원리 이해
- 비트 연산과 시프트 연산의 작동 방식 습득
- 데이터 손실이 발생하는 상황과 그 원인 파악
- 실무에서 안전한 타입 변환 방법 적용
